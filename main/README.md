# TSU Lecture Bot

Telegram-бот для автоматического подключения к онлайн-лекциям групп `932401`, `932402`, `932403`, `932404`.

Пользователь вводит ФИО и выбирает группу. Поддерживаемые предметы зафиксированы в коде, расписание подтягивается из TSU.InTime отдельно для выбранной группы, а подключение идёт как гость в e-class.

## Поддерживаемые группы

- `932401`
- `932402`
- `932403`
- `932404`

## Поддерживаемые предметы

- `Алгоритмы и структуры данных`
- `Методы оптимизации и исследование операций`
- `Архитектура вычислительных систем`

Предметы для этих групп одинаковые, но расписание берётся отдельно по `group_id` каждой группы.

## Запуск

```cmd
cd main
venv\Scripts\activate
python bot.py
```

Если окружение создаётся заново:

```cmd
py -3.12 -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
playwright install chromium
```

## `.env`

```env
BOT_TOKEN=123456:telegram-token
HEADLESS=true

SCHEDULE_LOOKAHEAD_DAYS=14
SCHEDULE_SYNC_INTERVAL_MINUTES=30
CHAT_EXIT_MESSAGE_THRESHOLD=5
CHAT_MONITOR_INTERVAL_SECONDS=15
```

`HEADLESS_MODE=true` тоже поддерживается для совместимости со старым `.env`.

## Где добавить ссылки

Группы и ссылки лежат в [group_presets.py](group_presets.py):

- `SUPPORTED_GROUPS` содержит `group_id` для каждой группы.
- `COMMON_LECTURES` содержит общие предметы и e-class ссылки.

В ссылках лучше оставлять комнату без ФИО:

```text
https://e-class.tsu.ru/#join:t3d870347-febe-45ab-929e-b98747aa494e,true
```

Бот сам подставит сохранённое ФИО пользователя.

## Как работает бот

1. Пользователь отправляет `/start`.
2. Вводит ФИО.
3. Выбирает одну из групп `932401`, `932402`, `932403`, `932404`.
4. Бот синхронизирует расписание TSU.InTime именно для выбранной группы.
5. Если в расписании есть поддерживаемые онлайн-лекции, бот планирует подключение.
6. В момент начала пары бот заходит в e-class гостем с сохранённым ФИО.

Ручное добавление лекций отключено: бот работает только с расписанием TSU.InTime и заранее заданными ссылками.

## Досрочный выход

После подключения бот пытается открыть чат e-class/BBB и каждые `CHAT_MONITOR_INTERVAL_SECONDS` секунд считает видимые сообщения. Если сообщений `>= CHAT_EXIT_MESSAGE_THRESHOLD`, бот закрывает вкладку и выходит из лекции досрочно.

Чтобы отключить этот режим:

```env
CHAT_EXIT_MESSAGE_THRESHOLD=0
```
