# TSU Lecture Bot

Telegram-бот для автоматического подключения к онлайн-лекциям Томского государственного университета.

Бот получает расписание из **TSU.InTime**, определяет поддерживаемые онлайн-лекции для выбранной группы и автоматически подключается к ним в **e-class** в качестве гостя.

## Возможности

- регистрация пользователя через Telegram;
- сохранение ФИО пользователя;
- выбор учебной группы;
- автоматическая синхронизация расписания из TSU.InTime;
- подключение к онлайн-лекциям в e-class;
- работа только с заранее заданными предметами и ссылками;
- досрочный выход из лекции по количеству сообщений в чате;
- поддержка запуска в headless-режиме через Playwright.

## Поддерживаемые группы

Бот поддерживает следующие группы:

- `932401`
- `932402`
- `932403`
- `932404`

Расписание загружается отдельно для каждой группы по её `group_id`.


## Как это работает

1. Пользователь отправляет команду `/start`.
2. Бот просит ввести ФИО.
3. Пользователь выбирает одну из поддерживаемых групп.
4. Бот синхронизирует расписание TSU.InTime для выбранной группы.
5. Если в расписании есть поддерживаемая онлайн-лекция, бот планирует подключение.
6. В момент начала пары бот открывает e-class и заходит как гость.
7. Для входа используется сохранённое ФИО пользователя.

Ручное добавление лекций отключено. Бот работает только с расписанием TSU.InTime и заранее заданными ссылками на e-class.


## Где настраиваются группы и ссылки

Группы и ссылки на лекции находятся в файле:

```text
main/group_presets.py
```

В этом файле используются две основные структуры:

| Структура | Назначение |
| --- | --- |
| `SUPPORTED_GROUPS` | Список поддерживаемых групп и их `group_id` |
| `COMMON_LECTURES` | Общие предметы и ссылки на e-class |

Пример ссылки на e-class:

```text
https://e-class.tsu.ru/#join:t3d870347-febe-45ab-929e-b98747aa494e,true
```

В ссылке лучше оставлять комнату без ФИО.  
Бот сам подставит сохранённое ФИО пользователя при подключении.

## Досрочный выход из лекции

После подключения бот пытается открыть чат e-class / BBB и проверяет количество видимых сообщений.

Проверка выполняется каждые:

```env
CHAT_MONITOR_INTERVAL_SECONDS=15
```

Если количество сообщений становится больше или равно значению:

```env
CHAT_EXIT_MESSAGE_THRESHOLD=5
```

бот закрывает вкладку и выходит из лекции досрочно.

Чтобы отключить досрочный выход, установите:

```env
CHAT_EXIT_MESSAGE_THRESHOLD=0
```

## Структура проекта

| Файл | Назначение |
| --- | --- |
| `main/bot.py` | Telegram-команды и основной сценарий работы |
| `main/group_presets.py` | Поддерживаемые группы и ссылки на e-class |
| `main/scheduler.py` | Синхронизация расписания и планирование подключений |
| `main/browser.py` | Вход в e-class и мониторинг чата |
| `main/database.py` | SQLite-база: пользователи и найденные занятия |
| `main/tsu_schedule.py` | Парсинг расписания TSU.InTime |
| `main/requirements.txt` | Python-зависимости проекта |
| `main/.env` | Локальная конфигурация проекта |

- Поддерживаемые предметы и ссылки задаются вручную в коде.
